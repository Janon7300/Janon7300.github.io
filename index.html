โหมดไม่ใช้ CBSE (Monolithic):
แก้ไข Navbar ต้องเปิดไฟล์เดียวกับส่วนอื่น ผลกระทบคาดเดายาก หากเกิดข้อผิดพลาดอาจทำให้พังทั้งหน้าเว็บ

จำลองการทำงาน 6 คุณลักษณะ (CBSE)

การ์ดที่ 1: Standardized (มีมาตรฐาน)

ใช้เทคโนโลยีที่เป็นมาตรฐาน ทำให้เบราว์เซอร์หรือระบบอื่นๆ สามารถรู้จักและทำงานร่วมกันได้ทันที

การ์ดที่ 2: Independent (เป็นอิสระ)

คอมโพเนนต์ทำงานด้วยตัวเอง ไม่แชร์ข้อมูล (State) มั่วซ้ายขวา ลองกดบวกเลขดู จะเห็นว่ามันไม่กวนกัน

ตัวนับ A

0

ตัวนับ B

0

การ์ดที่ 3: Composable (ประกอบกันได้)

เหมือนตัวต่อเลโก้ เราสามารถนำชิ้นส่วนต่างๆ มาประกอบกันเพื่อสร้างหน้าเว็บที่สมบูรณ์

(พื้นที่ประกอบร่าง)

การ์ดที่ 4: Deployable (ติดตั้ง/ใช้งานได้ทันที)

คอมโพเนนต์ถูกแพ็กเกจมาอย่างดี สามารถกดรันหรือติดตั้งใช้งานในระบบใหม่ได้เลยโดยไม่ต้องแก้ไขโค้ด

import { showBanner } from 'ui-lib';
showBanner("ระบบพร้อมใช้งาน!");

การ์ดที่ 5: Documented (มีข้อกำหนดชัดเจน)

คอมโพเนนต์จะมี Interface/เอกสารระบุชัดเจนว่าต้องการค่าอะไร ลองไม่กรอกราคาดูสิ ระบบจะปฏิเสธ!

การ์ดที่ 6: Encapsulated (กล่องดำ/ปกปิดความลับ)

คุณส่งแค่ "ยอดเงินตั้งต้น" คอมโพเนนต์จะแอบไปคำนวณภาษี 7% และค่าธรรมเนียมลับให้เอง โดยคุณไม่ต้องเห็นสูตร
